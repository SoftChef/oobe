流程處理

$born

初始化資料，觸發會使狀態$ready為true且並發送event。

sprite.$born(params) => self
Param Type Description
params? object Origin data

$toOrigin

回傳Origin的資料。

sprite.$toOrigin() => any

$reset

將屬性資料切換成宣告$born前的狀態。

sprite.$reset() => null

$setError

設定Sprite為錯誤並加入錯誤訊息且發送event。

sprite.$setError(error) => null
Param Type Description
error? any Error message

$reborn

初始化所有body和refs的資料並重新觸發born，且重新呼叫created行為並發送event。

通常我們不建議使用reborn行為，但有時ref的對象需要重新賦予數值，而其嚴格的檢查特性礙於執行此行為，因此才需要藉由reborn設定。

sprite.$reborn(params) => self
Param Type Description
params? object Origin data

操作處理

在使用雙向綁定的表單操作狀態的時候，容易造成修改同一對象導致全局狀態被改變，這類情境常發生在彈出式表單，例如以下例子：

<template>
    <!--當你修改這個值得同時store的狀態會被改變-->
    <input v-model="user.name">
</template>

<script>
import { mapGetters } from 'vuex'
export default {
    computed: {
        ...mapGetters: {
            user: 'user'
        }
    }
}
</script>

在這類操作的情境下，通常會把user物件分離出來，直到修改完畢再進行commit的行為，來避免取消操作造成的永久性的狀態停留。

$copy

觸發當下Sprite的$toOrigin()並用回傳的資料實例化一個新的Sprite。

sprite.$copy(options) => sprite
Param Type Description
options? object Sprite make options
options.save? boolean save
雖然可以直接複製對象進行修改，但反覆尋找修改對象並做資料轉換是個繁雜的行為，因此誕生了以下操作方法：

$out

$copy本Sprite並綁定目標，能使操作$revive時了解更動對象。

被宣告$out的對象會進入dead的狀態，本狀態會使數值無法被修改。

sprite.$out() => sprite

$revive

把修改資料寫入$out並回傳宣告的對象：

被宣告$revive的對象會進入dead的狀態，本狀態會使數值無法被修改。

sprite.$revive() => sprite

$dead

放棄本次的修改並回傳宣告的對象：

sprite.$dead() => sprite

$live

該Sprite是否進入dead的狀態，是為false。

sprite.$live => Boolean

最佳實踐

<template>
    <div v-if="targetUser && targetUser.$live">
        <input v-model="targetUser.name">
        <button @click.stop="save">save</button>
    </div>
</template>

<script>
import { mapGetters } from 'vuex'
export default {
    data() {
        return {
            targetUser: null
        }
    },
    computed: {
        ...mapGetters: {
            user: 'user'
        }
    },
    mounted() {
        this.targetUser = this.user.$out()
    },
    methods: {
        save() {
            // 經過targetUser修改的值會複寫回user
            this.targetUser.$revive()
        }
    },
    destroyed() {
        if (this.targetUser.$live) {
            this.targetUser.$dead()
        }
    }
}
</script>

視圖呈現

$ready

是否被呼叫過$born，有此狀態會回傳true。

sprite.$ready => Boolean

$dist

更改當前狀態，可更動狀態請見Dists。

$dist為$distortion方法的簡寫。

sprite.$dist(name) => self
Param Type Description
name string Distortion name.

$export

根據當前或指定狀態輸出值，詳情請見Dists。

sprite.$export(name, ...args) => any
Param Type Description
name? string 指定輸出狀態，省略則為當前狀態
...args any 輸出狀態可以接收參數

$show

指定數值是否不是fixed的狀態，詳情請見Dists。

sprite.$show(name) => Boolean
Param Type Description
name string Target

$isFixed

指定數值是否為fixed的狀態，詳情請見Dists。

sprite.$isFixed(name) => Boolean
Param Type Description
name string Target

$isHidden

指定數值是否為hidden的狀態，詳情請見Dists。

sprite.$isHidden(name) => Boolean
Param Type Description
name string Target

$isChange

當前數值是否與宣告$born時相同。

該方法在options.save宣告為false時無效。

sprite.$isChange(key) => Boolean
Param Type Description
key? string 可以只針對某一個屬性進行驗證

$validate

驗證屬性使用與Rule的規則相同。

sprite.$validate() => {
    success: Boolean,
    result: {
        // 如果是屬性回傳驗證錯誤字串
        [key]: String,
        // 如果是Ref對象回傳validate物件
        [key]: {
            success: Boolean,
            result: Object
        }
    }
}

$validateBy

基於指定Key的驗證方法，正確者回傳ture否則回傳錯誤訊息。

本方法只能驗證body宣告的值。

sprite.$validateBy(key) => true || String
Param Type Description
key? string 可以只針對某一個屬性進行驗證

$rules

回傳驗證的方法。

sprite.$rules(name, extra) => Array<Function>
Param Type Description
name string 指定的屬性
extra? array 擴增的驗證方法

$v($views)

經過計算的唯讀屬性，詳情請見Views。

$v為$views屬性的簡寫。

sprite.$v => Object

$o($loaders)

狀態控制單元，詳情請見Loaders。

$o為$loaders屬性的簡寫。

sprite.$o => Object

$meg

語系顯示單元，詳情請見Locale。

sprite.$meg(name, value) => String
Param Type Description
name string Message name
value? any Message Params

$error

當你觸發了$setError此屬性會從null改成輸入值，並且回傳errorMessage的值。

sprite.$error => any

資料處理

$body

複製並回傳當下屬性(body)的狀態。

sprite.$body() => Object

$keys

回傳Body的key值。

並不包含Refs的key。

sprite.$keys() => Array<String>

$put

批次寫入Body屬性。

該方法也會對應到相對的Refs的對象。

sprite.$put() => self
exampale
let sprite = {
    body() {
        return {
            name: '',
            age: null
        }
    }
}
// ...
user.$born()
user.$put({
    name: 'steve',
    age: 20
})
console.log(user.age) // 20

$raw

回傳$born傳入的值。

若宣告options.save則無法使用此功能。

sprite.$raw(assign) => any
Param Type Description
assign? object 可以對回傳的值進行合成

$map

Map對象。

sprite.$map => Object

方法處理

$fn

使用Methods的接口。

sprite.$fn => Object

$bind

與JavaScript的Funcition Bind相似，目的是在function被引用時能將this指向Sprite。

sprite.$bind(name) => Function
Param Type Description
name string Method name.
exampale
let sprite = {
    // ...
    body() {
        return {
            name: 'steve'
        }
    },
    methods: {
        getName() {
            return this.name
        }
    }
    // ...
}
// ...
let getName = user.$bind('getName')
console.log(getName()) // steve

事件處理

$on

監聽一則Event。

sprite.$on(eventName, callback) => { off: Function, id: String }
Param Type Description
eventName string Event name.
callback function Event callback.

$off

關閉監聽事件。

sprite.$off(eventName, id) => null
Param Type Description
eventName string Event name.
id string Listener id.

$onOnce

監聽的事件只觸發一次。

sprite.$onOnce(eventName, callback) => { off: Function, id: String }
Param Type Description
eventName string Event name.
callback function Event callback.

$emit

發送事件。

sprite.$emit(eventName, ...args) => null
Param Type Description
eventName string Event name.
...args args Emit items.

其他

$helper

Helper的接口。

sprite.$helper => Helper

$toObject

將Sprite轉換並回傳成可視化的資料。

這是給予開發人員方便開發的功能。

sprite.$toObject() => Object
exampale
let sprite = {
    // ...
    body() {
        return {
            name: ''
        }
    },
    views: {
        name() {
            return this.name
        }
    }
    // ...
}
// ...
let user = oobe.make('...', 'sprite', { name: 'steve' })
let result = user.$toOject()
console.log(result.$views.name) // steve
user.name = 'dave'
console.log(result.$views.name) // steve

$configs

該對象會指向Container的configs。

sprite.$configs => Object

$distName

當下的distortion的名稱。

sprite.$distName => String

$parent

如果該Sprite是來自Refs的參照對象時指向其參照者。

sprite.$parent => Sprite
exampale
let sprite = {
    // ...
    body() {
        return {
            name: 'steve'
        }
    },
    refs: {
        'sprite2': 'sprite2' 
    }
    // ...
}
let sprite2 = {
    // ...
}
let container = {
    sprites: {
        sprite,
        sprite2
    }
}
// ...
let user = oobe.make('...', 'sprite')
console.log(user.sprite2.$parent.name) // 'steve'

$self

回傳Self對象，會在$born宣告後產生。

sprite.$self => Object

$utils

回傳Container的utils對象。

sprite.$utils => Object
